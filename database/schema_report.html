📡 协议支持对比

🔐 CCC (Car Connectivity Consortium)

  • 8个槽位 slot_identifier (0-7)
  • 8字节 key_identifier
  • 16字节 device_identifier
  • X.509 证书链认证
  • SE芯片 强制安全芯片

📱 ICCOA (智惠车联开放联盟)

  • 无限 软件槽位
  • 手机号 phone_number
  • Token device_token
  • 密钥 对称密钥认证
  • TEE 可信执行环境

🔑 ICCE (智惠车联生态)

  • Key ID icce_key_identifier
  • 计数器 防重放保护
  • 密钥对 vehicle_pk / device_sk
  • 可选 PKI或密钥
  • TEE/SE 灵活配置

📂 核心数据表详解

1. 用户与车辆管理

表名 主键 核心字段 说明
users UUID email, password_hash, name 用户账户基础信息
vehicles VIN model, brand, support_ccc/iccoa/icce 车辆主档，协议支持标记
vehicle_aliases UUID vin, user_id, alias_name 用户自定义车辆别名

2. 数字钥匙管理

表名 协议 核心字段 说明
ccc_slots CCC slot_identifier(0-7), key_identifier CCC 8槽位管理
iccoa_keys ICCOA phone_number, auth_key, permission_mask ICCOA手机钥匙
icce_keys ICCE vehicle_pk, device_sk, counter ICCE公私钥对

3. 证书与安全

表名 证书类型 字段 说明
certificates X.509 cert_data, serial_number, valid_until 统一证书存储
devices 设备 device_identifier, se_type, public_key 设备与SE管理
secure_elements SE se_unique_id, scp03_key_version SE050等安全芯片

🔗 关键关系说明

核心关系

  1. users → vehicles: 一个用户可拥有多辆车 (1:N)
  2. vehicles → ccc_slots/iccoa_keys/icce_keys: 一辆车可有多把钥匙 (1:N)
  3. devices → keys: 一个设备可绑定多把钥匙 (1:N)
  4. certificates → certificates: 证书链自引用 (1:N)

协议特定关系

  • CCC: 车辆→槽位(0-7)→证书链
  • ICCOA: 手机号→认证密钥→授权分享
  • ICCE: 公私钥对→计数器→防重放

唯一索引

CREATE UNIQUE INDEX idx_vehicles_vin ON vehicles(vin); CREATE UNIQUE INDEX idx_devices_identifier ON devices(device_identifier); CREATE UNIQUE INDEX idx_ccc_key_identifier ON ccc_slots(key_identifier); CREATE UNIQUE INDEX idx_certs_hash ON certificates(cert_hash);

查询优化索引

-- 车辆查询 CREATE INDEX idx_vehicles_user ON vehicles(user_id); CREATE INDEX idx_vehicles_status ON vehicles(status); CREATE INDEX idx_vehicles_protocol ON vehicles(support_ccc, support_iccoa, support_icce); -- 钥匙查询 CREATE INDEX idx_ccc_slots_vin_state ON ccc_slots(vin, slot_state); CREATE INDEX idx_iccoa_phone ON iccoa_keys(phone_number); -- 日志查询 CREATE INDEX idx_logs_vin_time ON operation_logs(vin, created_at DESC);

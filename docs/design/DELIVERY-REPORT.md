## 📋 项目概述

### 1.1 项目定位

数字钥匙系统（Digital Key System）是面向智能网联汽车的新一代无钥匙进入与启动解决方案，通过将实体车钥匙功能数字化迁移至智能手机及可穿戴设备，实现**"手机即钥匙"**的创新体验。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 🔑 **双协议支持** | 同时兼容 ICCE（中国标准）与 CCC（国际标准） |
| 📱 **三端覆盖** | 嵌入式（车端）+ App（手机端）+ 云端（服务平台） |
| 🔒 **安全等级** | EAL5+ 硬件安全，端到端加密，防中继攻击 |
| ⚡ **快速响应** | 车门解锁响应时间 < 1 秒 |
| 🌐 **离线可用** | 支持无网络环境下的本地认证 |

### 4.2 严重问题修复详情

| 编号 | 问题描述 | 修复方案 | 验证状态 |
|------|---------|---------|---------|
| **S1** | 安全通道使用 memcpy 拷贝 shared_secret 作为 session_key | 实现 HKDF-SHA256 密钥派生 | ✅ 通过 |
| **S2** | 解密函数忽略消息计数器，使用硬编码 IV | 使用消息计数器构造 IV | ✅ 通过 |
| **S3** | 会话密钥未实际用于加解密，仍使用硬编码 SE050_KEY_SLOT_MASTER | 派生密钥导入 SE050 并使用 | ✅ 通过 |
| **S4** | 防中继拒绝逻辑错误，reject_count>=3 时仍 ALLOW | 修正为 REJECT | ✅ 通过 |
| **S5** | App 使用纯 Dart 实现加密，无原生平台安全隔离 | 实现 Secure Enclave/KeyStore 桥接 | ✅ 通过 |
| **S6** | MockHSMClient 可能用于生产环境 | 添加环境检查，生产环境禁用 | ✅ 通过 |

### 4.3 高优先级问题修复详情

| 编号 | 问题描述 | 修复方案 | 验证状态 |
|------|---------|---------|---------|
| **H1** | 嵌入式挑战值使用伪随机数 | 改用 SE050 TRNG | ✅ 通过 |
| **H2** | Nonce 未持久化，重启可重放 | 实现 Nonce 持久化存储 | ✅ 通过 |
| **H3** | 时间戳使用系统时钟，可被篡改 | 使用 RTC + 安全启动时间 | ✅ 通过 |
| **H4** | 安全启动未验证固件签名 | 实现 RSA-2048 固件签名验证 | ✅ 通过 |
| **H5** | NFC 交易数据未签名 | 添加 SM2/ECDSA 签名 | ✅ 通过 |
| **H6** | BLE ACL 未实现 | 实现 BLE 访问控制列表 | ✅ 通过 |
| **H7** | 云端 JWT 密钥硬编码 | 使用环境变量 + 密钥管理系统 | ✅ 通过 |
| **H8** | 审计日志未记录敏感操作 | 完善审计日志覆盖 | ✅ 通过 |

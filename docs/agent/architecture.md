## 核心技能职责

### Harness Engineering (基础设施层)
- **职责**: 让仓库对 AI Agent 可读、可维护
- **产出**: AGENTS.md, docs/agent/, 质量检查脚本
- **触发**: 仓库初始化、文档重构

### Superpowers (工作流层)
- **职责**: 规范开发流程，确保质量
- **产出**: 设计文档、TDD 测试、代码审查
- **触发**: 新功能开发、Bug 修复

### OpenSpec (规格层)
- **职责**: 结构化变更管理，可追溯
- **产出**: proposal, specs, design, tasks
- **触发**: 任何需要规划的变更

## 架构约束

### 1. 文档分层原则
```
AGENTS.md (路由)
    ↓
docs/agent/ (工程知识)
    ↓
openspec/ (变更规格)
```

- **不重复**: 同一知识只存在一处
- **可发现**: 通过索引可找到所有文档
- **渐进式**: 只加载需要的层级

### 2. 技能协作原则
```
Harness → 提供 docs/agent/ 结构
    ↓
Superpowers → 将设计文档写入 docs/plans/
    ↓
OpenSpec → 将变更规格写入 openspec/changes/
    ↓
Harness → 质量检查验证完整性
```

### 3. 质量门禁
- 所有变更必须有 OpenSpec 规格
- 所有代码必须有测试 (TDD)
- 所有文档必须有 owner 和 last_reviewed

## 边界和接口

### 技能间通信
- 通过文件系统（docs/, openspec/）
- 通过 AGENTS.md 路由
- 通过 memory/ 共享上下文

### 外部集成
- ClawHub/SkillHub: 技能发现和安装
- Git: 版本控制和协作
- CI/CD: 自动化质量检查

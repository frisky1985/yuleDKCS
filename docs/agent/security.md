# Security Requirements

本文档定义系统的安全边界、信任模型和安全实践。

## 安全边界

```
┌─────────────────────────────────────────────────────────────┐
│                    Trusted Zone                              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  Local Files (workspace)                             │    │
│  │  - AGENTS.md, docs/, openspec/                       │    │
│  │  - Skills in bundled/ and workspace/                 │    │
│  └─────────────────────────────────────────────────────┘    │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  Local Tools                                         │    │
│  │  - exec, read, write, edit                           │    │
│  │  - Git operations                                    │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ Gateway (approval required)
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    External Zone                             │
│  - Network requests (API calls)                             │
│  - Email/messaging                                          │
│  - Cloud services                                           │
│  - User data export                                         │
└─────────────────────────────────────────────────────────────┘
```

## 信任模型

### Trusted Sources
- 用户直接输入
- 本地文件系统
- Bundled skills（系统内置）
- Verified ClawHub/SkillHub skills

### Untrusted Sources
- 外部 API 响应
- 用户提供的链接内容
- 未验证的 skill 安装包

## 安全实践

### 1. 数据保护

| 数据类型 | 保护措施 |
|---------|---------|
| API Keys | 环境变量，不入 Git |
| 用户数据 | 本地存储，不外传 |
| 内存内容 | 仅限 main session |

### 2. 技能安全

```bash
# 安装前审查
clawhub inspect <skill-name>

# 检查可疑模式
# - 外部 API 调用
# - eval/exec 使用
# - 敏感数据访问
```

### 3. 执行安全

| 操作 | 安全级别 | 要求 |
|------|---------|------|
| 读取文件 | 低风险 | 自动允许 |
| 写入文件 | 中风险 | 工作区内自动允许 |
| 执行命令 | 高风险 | 需要批准 |
| 网络请求 | 高风险 | 需要批准 |
| 发送消息 | 高风险 | 需要批准 |

## 敏感数据处理

### 禁止行为

- 在日志中输出敏感数据
- 将 API keys 写入文件
- 在 group chat 中分享个人信息
- 将 MEMORY.md 内容暴露给非 main session

### 必须行为

- 使用环境变量存储 secrets
- 敏感数据加密存储
- 定期审计访问日志

## 审计和监控

### 审计日志

记录以下操作:
- 技能安装/卸载
- 外部网络请求
- 敏感数据访问
- 配置变更

### 异常检测

| 异常模式 | 处理 |
|---------|------|
| 大量数据导出 | 阻止 + 通知 |
| 未授权 API 调用 | 阻止 + 记录 |
| 可疑 skill 行为 | 禁用 + 报告 |

## 安全检查清单

### 技能安装前

- [ ] 检查作者可信度
- [ ] 审查代码内容
- [ ] 检查权限要求
- [ ] 验证来源 (ClawHub/SkillHub)

### 处理用户数据时

- [ ] 确认数据类型
- [ ] 检查目的地
- [ ] 获得用户明确同意
- [ ] 记录操作

### 外部请求前

- [ ] 验证 URL 安全性
- [ ] 检查请求内容
- [ ] 获得用户批准
- [ ] 记录请求和响应

## 安全事件响应

### 发现安全问题时

1. **立即行动**
   - 停止相关操作
   - 隔离受影响组件

2. **调查分析**
   - 收集日志
   - 确定影响范围
   - 找到根本原因

3. **修复验证**
   - 应用修复
   - 添加测试
   - 验证修复有效

4. **文档更新**
   - 更新安全文档
   - 记录教训
   - 通知相关人员

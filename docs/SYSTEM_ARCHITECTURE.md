## 1. 系统概述

### 1.1 产品定位

yuleDKCS 是一套完整的数字钥匙解决方案，支持 CCC、ICCOA、ICCE 三大主流协议，实现手机即钥匙的无钥匙进入和启动功能。

### 1.2 核心特性

- **多协议支持**: CCC 3.0 / ICCOA DK3.0&DK4.0 / ICCE
- **三模通信**: NFC + BLE + UWB
- **跨平台**: iOS / Android 双端 SDK
- **云端服务**: Go 高并发核心 + Java 协议适配器
- **车端嵌入式**: C 语言协议栈，支持多种 MCU

### 1.3 技术栈

| 层级 | 技术选型 |
|------|----------|
| 云端核心 | Go 1.22+ / gRPC / Kafka / PostgreSQL / Redis |
| 协议适配 | Java 17 / Spring Boot 3.2 |
| 手机端 | Kotlin (Android) / Swift (iOS) |
| 车端 | C / STM32L5 / NXP KW47A / NCJ29D6 / ST ST25R501 |
| 安全 | SE050 / TFM / AES-256-GCM / ECDSA P-256 |

## 5. 安全架构

### 5.1 密钥层级

```
┌─────────────────────────────────────────────────────────────┐
│                       密钥层级体系                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Level 0: Root Key (根密钥)                                 │
│  ├── 存储位置: SE050 安全芯片                               │
│  ├── 用途: 派生所有下级密钥                                 │
│  └── 保护: 物理隔离，无法读取                               │
│                                                             │
│  Level 1: Master Key (主密钥)                               │
│  ├── 存储位置: SE050                                        │
│  ├── 用途: 车辆级别密钥加密                                 │
│  └── 派生: HKDF(RootKey, "master")                         │
│                                                             │
│  Level 2: Device Key (设备密钥)                             │
│  ├── 存储位置: SE050 / 手机 Keychain                        │
│  ├── 用途: 设备身份认证                                     │
│  └── 派生: HKDF(MasterKey, device_id)                      │
│                                                             │
│  Level 3: Session Key (会话密钥)                            │
│  ├── 存储位置: RAM (运行时)                                 │
│  ├── 用途: 单次会话通信加密                                 │
│  └── 派生: ECDH(DeviceKey_Local, DeviceKey_Remote)         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 加密算法

| 用途 | 算法 | 密钥长度 |
|------|------|----------|
| 对称加密 | AES-256-GCM | 256 bit |
| 非对称加密 | ECDSA P-256 | 256 bit |
| 密钥协商 | ECDH P-256 | 256 bit |
| 密钥派生 | HKDF-SHA256 | - |
| 哈希 | SHA-256 | 256 bit |
| 签名 | ECDSA P-256 | 256 bit |

### 5.3 安全启动链

```
Boot ROM (不可变)
    │
    ▼ 验证签名
BootLoader (SE050 验证)
    │
    ▼ 验证签名
TFM (可信固件)
    │
    ▼ 验证签名
Application (应用固件)
    │
    ▼
运行时安全 (SE050 + TFM)
```

## 6. 通信协议

### 6.1 协议栈选择

| 场景 | 协议 | 理由 |
|------|------|------|
| 手机↔HUB | HTTPS/TLS 1.3 | 标准、安全、穿透性好 |
| HUB↔DKCS | gRPC 双向流 | 高性能、流式通信 |
| DKCS↔TCU | MQTT/TLS | 车联网标准、QoS 支持 |
| 手机↔TCU | BLE + UWB + NFC | 三模互补 |
| 内部数据 | BER-TLV | 紧凑、标准化 |

### 6.2 BER-TLV 编码规范

所有内部协议消息采用 BER-TLV 编码：

```
Tag (1-2 bytes) | Length (1-4 bytes) | Value (N bytes)

示例:
Tag: 0x80 (密钥ID)
Length: 0x10 (16字节)
Value: 0x01 0x23 0x45 ... (16字节密钥ID)
```

详见: `backend/cloud/protocol/encoding-rules.md`

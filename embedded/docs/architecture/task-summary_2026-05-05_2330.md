## 设计目标

基于 NXP KW47A (BLE)、NXP NCJ29D6 (UWB)、ST ST25R501 (NFC) 三颗芯片设计完整的数字钥匙产品系统架构。

## 关键结论

### 1. 系统架构
- 采用四层架构: 应用层 → 服务层 → 抽象层(HAL) → 驱动层
- 核心模块: AppService、CommMgr、SecMgr、PowerMgr
- 主状态机: 10个状态 (INIT → STANDBY → 检测/鉴权 → 解锁)

### 2. 通信架构
- **优先级策略**: NFC(应急) > UWB(近场) > BLE(数据)
- **距离触发**: <1m用NFC, 1-10m用UWB, >10m用BLE
- **状态切换**: 基于距离和事件的自动状态机

### 3. 安全架构
- **密钥方案**: SE050物理保护 + TFM可信执行环境
- **密钥类型**: Root Key / Master Key / Device Key / Session Key
- **加密算法**: AES-256-GCM, ECDSA P-256, HKDF, SHA-256
- **安全启动**: BootLoader → OS → App 三级签名验证

### 4. 电源管理
- **状态机**: ACTIVE → IDLE → SLEEP → DEEPSLEEP → OFF
- **功耗等级**: 15mA → 3mA → 100μA → 10μA → 1μA
- **唤醒源**: NFC场 / BLE广播 / UWB测距 / 按键 / 定时器

### 5. 车端接口
- **CAN预留**: 500Kbps, CAN 2.0B, ID 0x100-0x1FF
- **诊断**: SWD调试 + USB DFU升级
- **OTA**: BLE OTA + UWB辅助

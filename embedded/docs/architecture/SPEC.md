## 1. 系统概述

### 1.1 产品定位

数字钥匙是一款基于NFC/BLE/UWB三模通信的智能车辆解锁产品，支持无钥匙进入、一键启动、远程共享等功能。

### 1.2 硬件规格

| 组件 | 型号 | 备注 |
|------|------|------|
| BLE芯片 | NXP KW47A | 蓝牙5.0 Low Energy |
| UWB芯片 | NXP NCJ29D6 | 实时定位，穿墙能力强 |
| NFC芯片 | ST ST25R501 | 被动式通信，应急解锁 |
| MCU | STM32L5系列 | 安全MCU，TFM认证 |
| 安全芯片 | SE050 | 专用安全元件 |

### 3.2 通信优先级策略

| 优先级 | 模式 | 使用场景 |
|--------|------|----------|
| 1 (最高) | NFC | 应急解锁、备电解锁 |
| 2 | UWB | 精确距离判断、被动解锁 |
| 3 | BLE | 数据传输、远程控制 |

### 3.3 通信切换逻辑

```c
typedef enum {
    COMM_MODE_NFC = 0,
    COMM_MODE_UWB,
    COMM_MODE_BLE,
    COMM_MODE_IDLE
} CommMode_e;

// 通信模式自动切换状态机
typedef struct {
    CommMode_e current_mode;
    CommMode_e target_mode;
    uint8_t    uwb_distance;    // UWB测量的距离，单位:cm
    uint8_t    ble_rssi;        // BLE信号强度
    uint8_t   nfc_field_detect; // NFC场检测标志
} CommState_t;
```

### 5.2 低功耗策略

| 状态 | 功耗 | 唤醒条件 |
|------|------|----------|
| ACTIVE | <15mA | 任何事件 |

| DEEPSLEEP | <10μA | NFC场/按键 |
| POWEROFF | <1μA | 长按电源 |

| 唤醒源 | 唤醒方式 | 响应时间 |

| UWB测距 | 距离变化 | <200ms |
| 按键 | 机械触发 | <10ms |
| 定时器 | RTC唤醒 | 精确 |

### 5.4 电池优化策略

- **智能调频**: 根据任务动态调整工作频率
- **外设管理**: 不使用时关闭BLE/UWB/NFC
- **占空比优化**: BLE广播采用低占空比策略
- **批量处理**: 数据聚合后一次性传输

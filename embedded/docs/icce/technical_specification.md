# ICCE数字钥匙协议栈技术规格说明书

## 1. 概述

### 1.1 文档信息
- **项目名称**: ICCE数字钥匙协议栈
- **版本**: V1.0
- **日期**: 2026-05-05
- **状态**: 设计阶段

### 1.2 项目背景
ICCE(Intelligent Connected Car Enterprise)数字钥匙协议栈是基于NXP KW47A BLE芯片和边缘计算单元的车辆数字钥匙解决方案，实现手机即钥匙的功能。

### 1.3 目标
- 提供安全可靠的数字钥匙认证机制
- 支持BLE近场通信和边缘计算离线决策
- 满足车规级安全要求(ASIL-B)

### 2.2 硬件平台

#### 2.2.1 NXP KW47A BLE模块
- **内核**: ARM Cortex-M33 @ 96MHz
- **Flash**: 1MB
- **RAM**: 256KB
- **BLE**: BLE 5.3, 支持长距离(Coded PHY)
- **安全**: 支持TrustZone, AES-256, ECC
- **接口**: SPI, I2C, UART, CAN-FD

#### 2.2.2 边缘计算单元
- **处理器**: ARM Cortex-A53 @ 1.5GHz (四核)
- **内存**: 2GB DDR4
- **存储**: 16GB eMMC + SD卡扩展
- **安全**: 支持TEE(可信执行环境)
- **接口**: CAN-FD, Ethernet, USB

### 3.2 GATT服务定义

#### 3.2.1 数字钥匙服务 (UUID: 0x18XX)
| 特征 | UUID | 属性 | 描述 |
|------|------|------|------|
| 钥匙状态 | 0x2A01 | Read/Notify | 钥匙有效性、权限等级 |
| 测距数据 | 0x2A02 | Read/Notify | RSSI, ToF测距结果 |
| 认证挑战 | 0x2A03 | Write/Notify | 挑战-响应认证 |
| 控制命令 | 0x2A04 | Write | 解锁/启动命令 |
| 会话密钥 | 0x2A05 | Write | 加密会话密钥交换 |

#### 3.2.2 连接参数
- **广播间隔**: 100ms (快速) / 1s (慢速)
- **连接间隔**: 15ms (低延迟模式) / 100ms (省电模式)
- **从机延迟**: 0-4
- **超时**: 4000ms

### 3.3 安全机制

#### 3.3.1 配对方式
- **OOB配对**: 通过NFC或QR码传递公钥
- **数字密钥**: 符合Car Connectivity Consortium (CCC)标准
- **加密等级**: LE Secure Connections (AES-CCM)

#### 3.3.2 认证流程
```
手机端                          车端
  │                              │
  │◄────── 广播设备信息 ──────────│
  │                              │
  │────── 扫描并连接 ────────────►│
  │                              │
  │◄────── 发送挑战值(Nonce) ────│
  │                              │
  │────── 签名响应(PrivKey) ─────►│
  │                              │
  │◄────── 验证成功 + SessionKey │
  │                              │
  │────── 加密通信 ─────────────►│
  │                              │
```

### 4.2 缓存策略

#### 4.2.1 钥匙缓存
| 项目 | 缓存时间 | 更新策略 | 存储位置 |
|------|----------|----------|----------|
| 钥匙公钥 | 永久 | 云端推送 | 安全存储区 |
| 权限配置 | 24小时 | 定时同步 | 普通存储区 |
| 认证令牌 | 8小时 | 使用时刷新 | 内存 |
| 会话数据 | 会话周期 | 连接断开清除 | 内存 |

#### 4.2.2 离线策略
1. **完全离线模式**: 使用预置密钥进行本地认证
2. **弱网模式**: 缓存优先,后台同步
3. **在线模式**: 云端验证优先

### 4.3 离线决策逻辑

#### 4.3.1 决策规则
```
IF 钥匙在本地缓存 AND 
   签名验证通过 AND
   权限未过期 AND
   未达到风险阈值
THEN 授权成功
ELSE 拒绝并记录异常
```

#### 4.3.2 风险评估模型
- **设备指纹**: 验证设备特征
- **行为分析**: 检测异常操作模式
- **位置验证**: 可选的GPS辅助验证
- **时间窗口**: 限制操作频率

## 5. 车端集成规范

### 5.1 CAN消息定义

#### 5.1.1 数字钥匙消息 (ID: 0x7XX)
| 字节 | 内容 | 描述 |
|------|------|------|
| 0 | 命令类型 | 0x01:解锁 0x02:启动 0x03:查询 |
| 1-4 | 用户ID | 授权用户标识 |
| 5-7 | 时间戳 | 操作时间 |
| 8 | 校验和 | 数据完整性校验 |

#### 5.1.2 车辆状态消息 (ID: 0x8XX)
| 字节 | 内容 | 描述 |
|------|------|------|
| 0 | 状态标志 | 锁止状态、启动状态等 |
| 1-2 | 电池电压 | 单位: mV |
| 3 | 车门状态 | 各车门开关状态 |
| 4-8 | 扩展数据 | 预留 |

### 5.3 异常处理

#### 5.3.1 异常类型
| 类型 | 错误码 | 处理策略 |
|------|--------|----------|
| 认证失败 | 0x01-0x0F | 记录日志，延迟重试 |
| 通信超时 | 0x10-0x1F | 重连，降级模式 |
| 权限不足 | 0x20-0x2F | 拒绝操作，通知云端 |
| 系统故障 | 0x30-0x3F | 安全模式，限制功能 |
| 篡改检测 | 0xF0-0xFF | 立即锁止，安全告警 |

#### 5.3.2 故障恢复
1. **自动恢复**: 重启服务，重置状态
2. **降级运行**: 限制功能，保持基本安全
3. **安全模式**: 仅允许物理钥匙

## 6. 安全要求

### 6.1 密码学要求
- **密钥长度**: ECC P-256 或以上
- **对称加密**: AES-256-GCM
- **哈希算法**: SHA-256
- **随机数**: TRNG真随机数生成器

### 6.2 安全等级
| 资产 | 安全等级 | 保护措施 |
|------|----------|----------|
| 主密钥 | ASIL-D | HSM存储，永不导出 |
| 会话密钥 | ASIL-B | TEE保护，定期更新 |
| 用户数据 | ASIL-A | 加密存储 |
| 日志数据 | QM | 完整性保护 |

### 6.3 防护机制
- **防重放**: 挑战-响应机制，Nonce一次性使用
- **防中间人**: 证书双向认证
- **防降级**: 版本强制检查
- **防克隆**: 设备唯一标识绑定

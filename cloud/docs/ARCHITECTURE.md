## 2. 核心设计原则

### 2.1 HUB模式 - 多协议统一网关
- **适配器模式**: 每家手机厂商一个适配器，实现统一接口
- **协议转换**: CCC/ICCOA/ICCE → 内部统一模型 → 私有协议 → DKCS
- **热插拔**: 新增手机厂商只需加适配器，无需改核心逻辑
- **版本兼容**: 适配器内处理同一厂商不同版本协议差异

### 2.2 高并发设计
- 连接池: Redis连接池 + MySQL连接池 + gRPC连接池
- 异步队列: 密钥操作通过Kafka/RabbitMQ异步处理
- 缓存策略: 热点数据Redis缓存，设备状态本地缓存
- 限流熔断: 令牌桶限流 + 熔断器模式
- 水平扩展: 无状态服务 + K8s HPA自动扩缩容

### 2.3 数据安全
- 传输加密: TLS 1.3 + 双向认证
- 存储加密: 密钥数据AES-256-GCM加密存储
- HSM集成: 密钥操作通过HSM/SoftHSM执行
- 审计追踪: 全链路操作日志，不可篡改

## 3. 技术选型

| 组件 | 技术 | 理由 |
|------|------|------|
| 云服务 | Go 1.22+ | 高并发、低延迟、跨平台 |
| HUB网关 | Go + gRPC | 跨语言协议转换、流式通信 |
| 数据库 | MySQL 8.0 + Redis 7 | ACID + 高性能缓存 |
| 消息队列 | Kafka | 高吞吐、持久化、消费者组 |
| 容器 | Docker + K8s | 弹性伸缩 |
| 手机端 | Kotlin + Android 12+ | NFC HCE + UWB API |

## 4. 数据流

### 4.1 密钥绑定流程
```
手机App → BLE/NFC → 车端协议栈 → 车端生成密钥对
                                    ↓ MQTT
                              DKCS接收绑定请求
                                    ↓ gRPC
                              HUB协议转换
                                    ↓ HTTPS
                              手机厂商云端确认
                                    ↓ Push
                              手机App存储密钥
```

### 4.2 远程控车流程
```
手机App → HTTPS → HUB统一API → gRPC(私有协议) → DKCS → MQTT → TCU → CAN
```

### 4.3 密钥分享流程
```
车主App → HUB API → 生成分享码 → 接收方App → HUB验证 → DKCS下发 → 车端写入
```

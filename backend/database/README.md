> **支持协议**: CCC / ICCOA / ICCE

## 两套数据库的区别

### 1. 车云一体化业务数据库 (核心业务)

**文件**: `schema_v1.0.0.sql`

**用途**: 支撑数字钥匙车云一体化产品的核心业务

**核心表**:
| 表名 | 说明 |
|------|------|
| `users` | 用户账户管理 |
| `vehicles` | 车辆信息 (VIN主键) |
| `devices` | 手机设备管理 |
| `ccc_slots` | CCC数字钥匙槽位 |
| `iccoa_keys` | ICCOA密钥管理 |
| `icce_keys` | ICCE密钥管理 |
| `secure_elements` | SE安全元管理 |
| `certificates` | X.509证书管理 |
| `operation_logs` | 操作日志 (分区表) |
| `vehicle_telemetry` | 车辆遥测数据 |

**特点**:
- 支持多协议 (CCC/ICCOA/ICCE)
- 按月分区的日志表
- 完整的证书链管理
- 车辆-设备-密钥关联关系

---

### 2. 合规检测系统数据库 (子系统)

**目录**: `compliance_system/`

**用途**: 用于检测数字钥匙产品是否符合规范要求

**核心表**:
| 表名 | 说明 |
|------|------|
| `compliance_rules` | 合规规则定义 |
| `test_targets` | 测试对象(DUT) |
| `target_certificates` | 测试对象证书 |
| `test_suites` | 测试套件 |
| `test_tasks` | 测试任务实例 |
| `test_results` | 测试结果 (分区表) |
| `packet_captures` | 通信抓包记录 |
| `communication_flows` | 通信流水线 |
| `compliance_reports` | 合规报告 |

**特点**:
- 规则驱动的测试框架
- 支持自动化测试执行
- 抓包数据与测试结果关联
- 可生成签名的合规报告

## 快速开始

### 部署业务数据库

```bash
# 1. 连接数据库
psql -U dkcs_user -d dkcs_db

# 2. 执行业务数据库 Schema
\i schema_v1.0.0.sql

# 3. 检查表结构
\dt
```

### 部署合规检测数据库 (可选)

```bash
# 如果需要合规检测功能
cd compliance_system
psql -U dkcs_user -d dkcs_compliance_db
\i schema_v1.0.0.sql
```

# 数字钥匙云+手机端开发总结

## 整体架构

```
手机App ──HTTPS──→ HUB ──gRPC(私有协议)──→ DKCS ──MQTT──→ TCU
  │                  │                         │
  │ NFC/BLE/UWB      │ 协议适配               │ TSP对接
  │ (直连车端)        │ CCC/ICCOA/ICCE        │
  ↓                  ↓                         ↓
车端协议栈        手机厂商云端              车厂TSP系统
```

## 云端产出

### 1. 数据库 (schema.sql - 18KB)
9张核心表：dk_user, dk_device, dk_vehicle, dk_digital_key, dk_key_share, dk_vehicle_status, dk_control_command, dk_hub_adapter, dk_hub_route, dk_audit_log, dk_ota_campaign, dk_ota_progress

### 2. HUB (Go微服务)
- `api/v1/hub.proto` - gRPC协议定义 (4个service, 20+ RPC)
- `cmd/hub/main.go` - 启动入口
- `internal/adapter/registry.go` - 适配器注册中心 (热插拔)
- `internal/adapter/ccc_adapter.go` - CCC适配 (Apple/Samsung)
- `internal/adapter/iccoa_adapter.go` - ICCOA适配 (小米/OPPO/vivo)
- `internal/adapter/icce_adapter.go` - ICCE适配 (华为)
- `internal/service/key_management.go` - 密钥管理服务
- `internal/service/key_share.go` - 密钥分享服务
- `internal/service/hub_transport.go` - 私有协议转发
- `internal/service/vehicle_control.go` - 车辆控制
- `internal/gateway/rest_gateway.go` - REST→gRPC网关

### 3. DKCS (Go微服务 - 车厂部署)
- `cmd/dkcs/main.go` - 启动入口 (mTLS连接HUB)
- `internal/keymgmt/service.go` - 密钥管理
- `internal/tsp/service.go` - TSP对接
- `internal/device/service.go` - 设备管理

### 4. 部署
- `deploy/docker-compose.yml` - MySQL+Redis+Kafka+EMQX+HUB+DKCS

## 手机端产出

### Android App (Kotlin)
- `Models.kt` - 数据模型 (协议/密钥/权限/分享)
- `ble/BleManager.kt` - BLE通信 (自动协议识别+GATT)
- `nfc/NfcManager.kt` - NFC读写+配对+认证
- `uwb/UwbManager.kt` - UWB FiRa测距+分区触发
- `api/HubApiClient.kt` - 云端API客户端
- `se/SecureElementManager.kt` - SE密钥管理 (Android Keystore/StrongBox)
- `ui/KeyWalletActivity.kt` - 钱包主界面 (本地+远程控制)

## HUB架构亮点

1. **适配器模式**: 新增手机厂商只需加adapter，不改核心
2. **协议自动转换**: CCC/ICCOA/ICCE → 统一模型 → 私有协议
3. **跨厂商分享**: ICCOA联盟内密钥互认
4. **高并发**: 连接池+Kafka异步+Redis缓存+K8s HPA
5. **安全**: mTLS双向认证+HSM密钥操作+审计追踪
